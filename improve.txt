继续的话，我建议你别再试“往特征后面拼几何先验”了，近两年更值得借鉴的模块可以分成两类。

**最适合你当前这份无训练 `Simple3D` 的**
1. `Joint Local-Global Memory Bank`
来源：PointCore, 2025，[ScienceDirect](https://www.sciencedirect.com/science/article/abs/pii/S0893608025013279)  
适合接入：
- [FPFH.py](/D:/MiniShift-Simple3D-main/feature_extractors/FPFH.py) 输出一个全局 descriptor
- [features.py](/D:/MiniShift-Simple3D-main/feature_extractors/features.py) 里把局部库和全局库联合匹配  
模块价值：
- 比你试过的“拼接曲率/上下文”更像真正的新模块
- 更适合 memory-bank 框架
- 不一定需要训练  
优先级：最高

2. `Ranking-based Score Normalization`
来源：PointCore, 2025，[ScienceDirect](https://www.sciencedirect.com/science/article/abs/pii/S0893608025013279)  
适合接入：
- [features.py](/D:/MiniShift-Simple3D-main/feature_extractors/features.py) 的 `compute_anomay_scores()` 后  
模块价值：
- 如果你后面做 local/global 双分支，这个模块很重要
- 不需要训练
- 比单纯改 top-k pooling 更像可写进论文的模块  
优先级：高

3. `Registration-guided Rotation-Invariant Module`
来源：NeurIPS 2025，[OpenReview](https://openreview.net/forum?id=QynKUq9X3L)  
适合接入：
- 预处理/对齐模块，放在数据加载后、特征提取前  
模块价值：
- 对 ShapeNet 这类姿态变化更敏感的数据更有意义
- 比继续堆局部几何统计更可能有效  
缺点：
- 改动会比前两个大
- 实现成本中高  
优先级：中高

4. `Density-aware Upsampling / Homogeneity Module`
来源：DAUP, 2024，[ScienceDirect](https://www.sciencedirect.com/science/article/abs/pii/S1474034624004713)  
适合接入：
- `data/*.py` 或 [FPFH.py](/D:/MiniShift-Simple3D-main/feature_extractors/FPFH.py) 前  
模块价值：
- 从数据分布层面改善局部邻域质量
- 对高分辨率但密度不均的数据更合理  
缺点：
- 原论文是学习式 upsampling
- 你如果坚持无训练，只能借思路做简化版  
优先级：中

**如果你愿意接受训练模块**
5. `Feature Adaptor`
来源：Scientific Reports 2026，[Nature](https://www.nature.com/articles/s41598-026-41255-2)  
适合接入：
- [FPFH.py](/D:/MiniShift-Simple3D-main/feature_extractors/FPFH.py) 的 `agg_point_feature` 后  
模块价值：
- 两层 MLP 适配器，很轻
- 比你前面试的无训练增强更可能真正带来提升  
缺点：
- 需要训练  
优先级：很高

6. `Selective Anomalous Feature Generator`
来源：同上，[Nature](https://www.nature.com/articles/s41598-026-41255-2)  
适合接入：
- 特征级数据增强分支  
模块价值：
- 能制造 hard negative
- 适合提升判别边界  
缺点：
- 需要训练
- 会把框架改成训练式  
优先级：中高

7. `Point Offset Prediction Module`
来源：PO3AD, CVPR 2025，[CVPR Open Access](https://openaccess.thecvf.com/content/CVPR2025/html/Ye_PO3AD_Predicting_Point_Offsets_toward_Better_3D_Point_Cloud_Anomaly_CVPR_2025_paper.html)  
模块价值：
- 是近两年很强的一条线
- 正常向量引导伪异常、点偏移预测都很有论文感  
缺点：
- 基本等于另起一套训练框架
- 不适合在当前 Simple3D 上做“小改”  
优先级：如果你接受重构，则高；否则低

**结合你现在的实验结果，我的建议非常明确**
如果你还想尽量保持当前框架不训练，下一步最值得试的是：

1. `Joint Local-Global Memory Bank`
2. `Ranking-based Score Normalization`

如果你愿意开始引入训练，最值得试的是：

1. `Feature Adaptor`
2. 再考虑 `Selective Anomalous Feature Generator`

一句话总结：
- 无训练路线：优先做 `PointCore` 这条线
- 可训练路线：优先做 `Feature Adaptor`

我建议先做：**Joint Local-Global Memory Bank**。




1. 边界感知的 point-level refinement
最推荐，性价比最高。

思路：

对每个点看它周围点的：
空间距离
法向差异
分数差异
在几何连续区域内平滑分数
在边界/高曲率/法向突变处抑制平滑
效果目标：

减少噪点
保住异常边界
提升 pixel-level
这会比继续调核函数更直接。

2. patch-to-point 插值方式升级
你现在的问题很可能有一部分就出在这里。

可以把当前比较粗的插值，改成：

距离加权
法向一致性加权
异常不确定性加权
也就是：
不是所有 patch 对 point 的贡献都一样。

这一步很适合你当前代码结构，也很可能直接改善 P-ROC。

3. point-level uncertainty gate
你现在 gate 是 patch-level 的。
后面可以进一步做：

patch-level 先给出 coarse anomaly
point-level 再根据局部不确定性决定修正强度
这比继续全局统一调 gate 更贴近最终目标。




方案 A：Point-level Geometric Refinement
在 s_map 之后做：

空间邻域聚合
法向一致性约束
分数边界保护
这更像“几何引导的点级细化”。

方案 B：Pseudo-offset inspired correction
不用训练 offset 网络，而是用局部几何统计近似“偏移程度”：

点到局部切平面距离
点到邻域均值位置偏差
法向不一致程度
再把这些量作为 point-level refinement 的权重。

这样你借到了 PO3AD 的核心思想：
异常点偏离正常局部结构
但不必把整个训练框架搬进来。

如果你愿意接受训练
那当然可以进一步往真正的 PO3AD 路线走：

加一个轻量 point offset head
用正常样本 + 伪异常训练
再把输出接到当前 point-level score refinement




方案 1：在 patch center 上做 refinement
把当前 point refine 的输入从：

unorganized_pc_no_zeros
s_map
改成：

center
patch_scores
流程变成：

对 patch center 做 KNN
在 patch center 邻域上估计局部几何
计算 patch-level refinement map
修正 patch_scores
再像现在一样插值成 s_map
优点
速度会快很多
和当前主线最一致
很可能还能保住你现在的收益
我最推荐这个
方案 2：只对高风险点做 point refine
也就是不是所有点都 refine，而是只 refine：

高 anomaly score 的点
或高 uncertainty 的点
或 top 20% 的候选异常点
优点
更省时间
更贴近“只修正需要修正的地方”
缺点
实现会比方案 1 稍复杂一点
方案 3：保留 point-level，但简化几何项
现在你同时用了：

plane deviation
normal inconsistency
可以先简化成只保留一个，比如：

只用 plane
或只用 normal
这样会少掉一部分计算。

优点
改动小
适合快速验证
缺点
提速有限，不如方案 1 明显
